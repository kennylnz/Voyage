VPN to enable access to your desktop at home from office

This is a quick note for how I sort out connection to my desktop at home from office.
The needs of connecting your desktop from office or anyway is a valid for many IT guys. There are many options, you might need use one suit your needs.
1 NAT thru your home router. Usually, you have a modem/router at home to connect to Internet. The modem will get a public IP address each time. What you need to do is setup a NAT port forwarding from the public ip to you desktop IP. Although you can hardly get a static public IP, as its from a pool managed by your broadband provider, there are options to map your domain dynamically to the dynamic ip you got each time. This is not a bad option, as it’s almost free.

2 Setup VPN server, and connect your desktop to the vpn server. It could be L2TP vpn server, or it could be OpenVPN server. both available from AWS cloud market place.

L2TP

L2TP is a protocol, so it’s widely supported, all operating system, mobile deives, So I decided to start with this one.
L2TP one was very friendly to windows machine, almost no complex settings, just get the necessary credentials and keys and set in windows VPN connection.
But my Desktop is a Unbuntu 18.04, actually there are a few posts tell how to setup L2TP connections, but unfortunately none of them suite Unbuntu 18.04, easy enough. Thus I gave up L2TP.

OpenVPN

OpenVpn instance requires some configurations during initialize, you need login to the EC2 instance. After that you can config from web UI. Couple of notes for my configurations.
1 static ip for certain user, which is needed for the desktop machine, this requires set a IP pool for static IPs first, it’s under VPN Settings, and then specify static ip for user under User Permissions.
2 you need manage the ACL per user, to specify the access to and access from to ensure the your laptop/mobile connection have access to your desktop (suppose to use static ip).
3 start vpn connection as service in linux, first you need enable auto-login for the user used by your desktop machine, and download the profile to linux machine.
in ubuntu, you need use systemctl to start the openvpn service.
